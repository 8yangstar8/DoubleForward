# Double Forward — 双人合作冒险游戏开发计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 开发一款 Android 平台的双人合作冒险游戏，玩家需要操控两个拥有不同能力的角色，通过协作解谜、闯关来推进游戏进程。

**Architecture:** 基于 Unity 引擎开发，采用 Host-Client 架构实现双人联机。游戏以 2.5D 横版视角呈现，每个关卡围绕"非对称能力门"设计——即必须两个玩家配合各自独特能力才能通过。网络层使用 Unity Netcode for GameObjects + Mirror 实现局域网和在线联机。

**Tech Stack:** Unity 2022 LTS / C# / Unity Netcode for GameObjects / Mirror Networking / Unity 2D Physics (Box2D) / ScriptableObject 数据驱动 / Photon Relay (NAT穿透备选)

## 2. 技术选型

### 2.1 游戏引擎：Unity 2022 LTS

**选择理由：**
- 成熟的 Android 构建管线，一键出包
- 丰富的 2D/2.5D 工具链（Tilemap、Sprite、2D Physics）
- 强大的网络多人框架生态（Netcode / Mirror / Photon）
- Asset Store 海量资源加速开发
- C# 语言开发效率高，调试方便

### 2.2 网络框架

| 方案 | 用途 | 说明 |
|------|------|------|
| **Unity Netcode for GameObjects** | 核心网络层 | Unity 官方方案，与引擎深度集成 |
| **Mirror** | 局域网备选 | 开源、轻量，适合 LAN 发现 |
| **Unity Relay + Lobby** | 在线联机 | 处理 NAT 穿透与房间匹配 |
| **Photon Fusion** | 备选在线方案 | 免费额度充足，2人场景足够 |

### 2.3 其他关键技术

| 技术 | 用途 |
|------|------|
| **Unity 2D Physics (Box2D)** | 角色移动、碰撞检测、物理谜题 |
| **Unity Tilemap** | 关卡地图编辑与渲染 |
| **ScriptableObject** | 关卡数据、角色属性、道具配置 |
| **DOTween** | UI/角色动画补间 |
| **Addressables** | 资源异步加载与内存管理 |
| **Unity Cloud Save** | 存档云同步 |
| **TextMeshPro** | UI 文字渲染 |

## 3. 游戏核心设计

### 3.1 核心玩法循环

```
发现障碍 → 观察环境 → 沟通协作 → 分别行动 → 能力互补 → 解开谜题 → 推进关卡
```

### 3.2 双角色设计

#### 角色 A：「光 · Lux」
| 属性 | 描述 |
|------|------|
| **核心能力** | 光束照射——照亮黑暗区域、激活光敏机关、折射光线 |
| **移动特点** | 轻盈跳跃，可二段跳 |
| **辅助能力** | 创建临时光桥（持续3秒），供队友通过 |
| **被动** | 在光照区域移速提升 20% |

#### 角色 B：「影 · Nox」
| 属性 | 描述 |
|------|------|
| **核心能力** | 影子潜行——穿过阴影墙壁、操控影子形态推动物体 |
| **移动特点** | 地面速度快，可冲刺穿过薄墙 |
| **辅助能力** | 创建阴影区域（持续3秒），阻挡敌人视线 |
| **被动** | 在阴影区域攻击力提升 20% |

### 3.3 合作机制

| 机制 | 描述 | 示例 |
|------|------|------|
| **能力互补门** | 需要两种能力同时/先后使用才能通过 | Lux 照亮通道 + Nox 在阴影区推动开关 |
| **双人压板** | 两人需同时站在不同压板上 | 同时踩下打开大门 |
| **传递谜题** | 一人获取物品传递给另一人使用 | Lux 折射光线到 Nox 需要照亮的位置 |
| **时序配合** | 按特定时间顺序执行操作 | 先创建阴影 → 再在阴影中激活机关 |
| **载体交互** | 一个角色作为另一个的平台/载具 | Nox 创建影子台阶，Lux 踩上去够到高处 |

### 3.4 操控方案 (移动端)

```
┌──────────────────────────────────────────────┐
│                                              │
│  ┌─────┐                        [技能B]      │
│  │     │                                     │
│  │摇杆 │                  [技能A]  [跳跃]     │
│  │     │                                     │
│  └─────┘                                     │
│   左手区域                      右手区域       │
└──────────────────────────────────────────────┘
```

- **左侧虚拟摇杆**：控制角色移动
- **右侧按钮区**：跳跃、主技能、副技能
- **本地分屏模式**：屏幕上下分割，各占一半

## 4. 系统架构

### 4.1 整体架构图

```
┌─────────────────────────────────────────────────────┐
│                    Game Manager                      │
│         (场景管理 / 游戏状态 / 存档系统)               │
├──────────┬──────────┬──────────┬────────────────────┤
│  Input   │ Player   │  Level   │   Network          │
│  System  │ System   │  System  │   Manager           │
│          │          │          │                     │
│ ·虚拟摇杆 │ ·角色控制 │ ·关卡加载 │ ·Host/Client       │
│ ·按钮映射 │ ·能力系统 │ ·谜题逻辑 │ ·状态同步           │
│ ·分屏输入 │ ·动画状态 │ ·机关系统 │ ·房间管理           │
│          │ ·物理交互 │ ·检查点  │ ·延迟补偿           │
├──────────┴──────────┴──────────┴────────────────────┤
│                    Unity Engine Layer                 │
│    2D Physics / Tilemap / Rendering / Audio          │
└─────────────────────────────────────────────────────┘
```

### 4.2 核心系统说明

#### GameManager (单例)
- 管理游戏全局状态（主菜单 / 游戏中 / 暂停 / 结算）
- 协调各子系统初始化与销毁
- 处理存档读写

#### InputSystem (输入系统)
- 基于 Unity New Input System
- 支持触屏虚拟摇杆 + 物理手柄
- 本地分屏时管理双玩家输入分离

#### PlayerSystem (玩家系统)
- 角色移动控制（地面移动、跳跃、二段跳、冲刺）
- 能力系统（主动技能 CD 管理、被动效果）
- 动画状态机（Idle / Run / Jump / Skill / Death）
- 碰撞与物理交互

#### LevelSystem (关卡系统)
- ScriptableObject 驱动的关卡数据
- 谜题组件（压板、开关、传送门、移动平台）
- 检查点与重生系统
- 关卡进度追踪

#### NetworkManager (网络管理)
- Host-Client 模型（一个玩家做主机）
- 角色位置/状态快照同步
- 谜题状态通过 RPC 事件同步
- 连接管理与断线重连

## 5. 关卡设计

### 5.1 世界观与主题

**故事背景：** Lux（光）和 Nox（影）原本是一体的存在，被神秘力量分离后散落在不同的世界碎片中。他们必须穿越五个风格迥异的世界，重新找到彼此的联系，最终合为一体。

### 5.2 关卡规划

| 章节 | 主题 | 核心机制 | 关卡数 | 预计时长 |
|------|------|----------|--------|----------|
| **第一章：觉醒森林** | 自然/教学 | 基础移动 + 简单压板配合 | 3 关 | 30 分钟 |
| **第二章：镜像工厂** | 蒸汽朋克 | 齿轮传动 + 光线折射谜题 | 4 关 | 50 分钟 |
| **第三章：深渊水域** | 海底洞穴 | 水流控制 + 光影切换 | 4 关 | 50 分钟 |
| **第四章：浮空遗迹** | 天空废墟 | 重力变换 + 时序配合 | 5 关 | 60 分钟 |
| **第五章：虚无核心** | 抽象空间 | 全能力综合 + Boss 战 | 4 关 | 50 分钟 |

### 5.3 关卡设计原则

1. **渐进复杂度**：每章前 1-2 关引入新机制，后续关卡组合使用
2. **非对称但平等**：两个角色的操作量和重要性大致相等
3. **失败成本低**：检查点密集，死亡后快速重生，不惩罚探索
4. **视觉引导**：用光影、颜色、粒子效果暗示可交互区域
5. **休息节奏**：每 2-3 个谜题后安排一段轻松的移动/探索段落

### 5.4 单关卡结构模板

```
┌─────────────────────────────────────────────┐
│  入口演出 (5s)                               │
│  ↓                                          │
│  教学/热身谜题 (简单，1个机制)                  │
│  ↓                                          │
│  [检查点 A]                                  │
│  ↓                                          │
│  主谜题 1 (中等，2个机制组合)                   │
│  ↓                                          │
│  探索/收集段落 (可选挑战)                      │
│  ↓                                          │
│  [检查点 B]                                  │
│  ↓                                          │
│  主谜题 2 (较难，多机制 + 时序)                 │
│  ↓                                          │
│  关卡结算 + 过场                              │
└─────────────────────────────────────────────┘
```

## 6. 网络联机方案

### 6.1 联机模式总览

| 模式 | 实现方案 | 延迟目标 | 优先级 |
|------|----------|----------|--------|
| **本地分屏** | 单设备上下分屏，无网络 | 0ms | P0 |
| **局域网联机** | Unity Netcode LAN Discovery | <20ms | P0 |
| **在线联机** | Unity Relay + Lobby | <100ms | P1 |

### 6.2 网络架构：Host-Client 模型

```
┌─────────────┐         ┌─────────────┐
│   Player 1  │ ◄─────► │   Player 2  │
│   (Host)    │  网络层  │   (Client)  │
│             │         │             │
│ ·游戏逻辑权威│         │ ·输入发送    │
│ ·物理模拟   │         │ ·状态插值    │
│ ·谜题判定   │         │ ·预测回滚    │
└─────────────┘         └─────────────┘
```

**选择 Host-Client 而非 dedicated server 的理由：**
- 仅 2 人合作，无竞技反作弊需求
- 降低运维成本，无需服务器
- 实现复杂度大幅降低

### 6.3 同步策略

| 数据类型 | 同步方式 | 频率 |
|----------|----------|------|
| 角色位置/速度 | NetworkTransform 快照插值 | 每帧 (30Hz) |
| 角色动画状态 | NetworkAnimator | 状态变化时 |
| 技能施放 | ServerRpc → ClientRpc | 事件触发 |
| 谜题/机关状态 | NetworkVariable | 状态变化时 |
| 关卡进度 | ServerRpc → 全局广播 | 事件触发 |

### 6.4 延迟补偿

- **客户端预测**：本地玩家立即响应输入，服务端验证后修正
- **快照插值**：远程玩家位置用 100ms 缓冲插值，保证平滑
- **事件可靠传输**：谜题状态变更使用可靠 RPC，保证一致性

### 6.5 房间流程

```
主菜单 → 选择联机模式
         ├─ 本地分屏 → 直接开始
         ├─ 局域网 → 创建/搜索房间 → 匹配 → 开始
         └─ 在线 → 创建/加入房间(邀请码) → Relay连接 → 开始
```

```csharp
using UnityEngine;

public class GameManager : MonoBehaviour
{
    public static GameManager Instance { get; private set; }

    public enum GameState { MainMenu, Loading, Playing, Paused, LevelComplete }
    public GameState CurrentState { get; private set; } = GameState.MainMenu;

    public event System.Action<GameState> OnGameStateChanged;

    void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }
        Instance = this;
        DontDestroyOnLoad(gameObject);
    }

    public void SetState(GameState newState)
    {
        if (CurrentState == newState) return;
        CurrentState = newState;
        OnGameStateChanged?.Invoke(newState);

        switch (newState)
        {
            case GameState.Paused:
                Time.timeScale = 0f;
                break;
            case GameState.Playing:
                Time.timeScale = 1f;
                break;
        }
    }
}
```

using UnityEngine;
using UnityEngine.EventSystems;

public class VirtualJoystick : MonoBehaviour, IPointerDownHandler, IDragHandler, IPointerUpHandler
{
    [SerializeField] private RectTransform knob;
    [SerializeField] private float maxRadius = 80f;

    public Vector2 Direction { get; private set; }

    private Vector2 center;

    public void OnPointerDown(PointerEventData eventData)
    {
        center = transform.position;
        OnDrag(eventData);
    }

    public void OnDrag(PointerEventData eventData)
    {
        Vector2 offset = eventData.position - center;
        offset = Vector2.ClampMagnitude(offset, maxRadius);
        knob.position = center + offset;
        Direction = offset / maxRadius;
    }

    public void OnPointerUp(PointerEventData eventData)
    {
        knob.localPosition = Vector2.zero;
        Direction = Vector2.zero;
    }
}
```

using UnityEngine;

[RequireComponent(typeof(Rigidbody2D))]
public class PlayerController : MonoBehaviour
{
    [Header("Movement")]
    [SerializeField] private float moveSpeed = 5f;
    [SerializeField] private float jumpForce = 10f;

    [Header("Ground Check")]
    [SerializeField] private Transform groundCheck;
    [SerializeField] private float groundCheckRadius = 0.2f;
    [SerializeField] private LayerMask groundLayer;

    private Rigidbody2D rb;
    private bool isGrounded;
    private int playerIndex;

    public void Initialize(int index) => playerIndex = index;

    void Awake() => rb = GetComponent<Rigidbody2D>();

    void Update()
    {
        isGrounded = Physics2D.OverlapCircle(groundCheck.position, groundCheckRadius, groundLayer);
    }

    public void SetMoveInput(Vector2 input)
    {
        rb.linearVelocity = new Vector2(input.x * moveSpeed, rb.linearVelocity.y);

        if (input.x != 0)
            transform.localScale = new Vector3(Mathf.Sign(input.x), 1, 1);
    }

    public void TryJump()
    {
        if (isGrounded)
            rb.linearVelocity = new Vector2(rb.linearVelocity.x, jumpForce);
    }
}
```

```csharp
using UnityEngine;

public abstract class PlayerAbilityBase : MonoBehaviour
{
    [SerializeField] protected float cooldown = 2f;
    protected float cooldownTimer;
    public bool IsReady => cooldownTimer <= 0f;

    protected virtual void Update()
    {
        if (cooldownTimer > 0f)
            cooldownTimer -= Time.deltaTime;
    }

    public void TryActivate()
    {
        if (!IsReady) return;
        Activate();
        cooldownTimer = cooldown;
    }

    protected abstract void Activate();
}
```

```csharp
using UnityEngine;

public class NoxAbilities : PlayerAbilityBase
{
    [SerializeField] private float dashDistance = 3f;
    [SerializeField] private float dashDuration = 0.2f;
    [SerializeField] private LayerMask shadowWallLayer;

    protected override void Activate()
    {
        StartCoroutine(DashThroughShadow());
    }

    private System.Collections.IEnumerator DashThroughShadow()
    {
        var rb = GetComponent<Rigidbody2D>();
        var collider = GetComponent<Collider2D>();
        float dir = transform.localScale.x;

        collider.excludeLayers |= shadowWallLayer;
        rb.linearVelocity = new Vector2(dir * dashDistance / dashDuration, 0);

        yield return new WaitForSeconds(dashDuration);

        collider.excludeLayers &= ~shadowWallLayer;
    }
}
```
